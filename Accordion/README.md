## 요구사항 정리 및 접근방법
여러 패널 죽 오직 하나만 열렬 수 있고, 같은 패널을 다시 클릭하면 닫혀야함. 
처음에는 각 accordion 내부에서 상태를 관리하려고 했지만, 그러면 각 컴포넌트가 서로의 열림상태를 모르는 구조가 되어 하나만 열기 조건을 보장하지 못함.
따라서 상태의 소유권을 재설계하고 열려 있는 패널은 전체에서 1개이므로, 공통 부모인 App에 openIndex를 두고 단일 소스로 만들었다.
Accordion.js는 stateless, openIndex === index 여부로 렌더링만 결정한다.
클릭 이벤트는 토글 규칙으로 이미 열려있으면 null로 바꿔닫고 아니면 해당 index로 바꿔서 연다.
